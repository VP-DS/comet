# AppHeader

The `AppHeader` component should be used as the main application header inside [`MasterLayout`](/story/docs-components-masterlayout--page).
It should contain the controls for the main-menu and other commonly needed action e.g. user-settings and logout.

_**Note:** When using the component inside `MasterLayout`, the props `position` and `headerHeight` should not be set, they may only be useful, if the component is used on it's own._

## Components

The AppHeader system consists of multiple basic components that can be combined to build a application-specific global header.

-   `AppHeader`
-   `AppHeaderMenuButton`
-   `AppHeaderFillSpace`
-   `AppHeaderAction`
-   `AppHeaderDropdown`


### AppHeader

`AppHeader` gives you a blank header, to which you can add things like a menu-button, a application-specific logo and actions to the `children`.

<Canvas>
    <Story id="stories-components-appheader-empty--appheader-empty" />

#### Props

| Name          | Type          | Default value                    | Description                                                                                          |
|:------------- |:------------- |:-------------------------------- |:---------------------------------------------------------------------------------------------------- |
| headerHeight  | number        |                                  | Sets the height of the `AppHeader`, should only be used if `AppHeader` is not inside `MasterLayout`. |

Props from [AppBar](https://material-ui.com/api/app-bar) are also used.

### AppHeaderMenuButton

`AppHeaderMenuButton` will give you the default menu-button.
Optionally you can add a custom icon as `children` or add a custom `onClick` if you want it to do something other than toggling the menu in `MasterLayout`.

<Canvas>
    <Story id="stories-components-appheader-menubutton--appheader-menubutton" />
</Canvas>

#### Props

| Name          | Type          | Default value                    | Description                                    |
|:------------- |:------------- |:-------------------------------- |:---------------------------------------------- |
| children      | node          | `<Hamburger fontSize="large" />` | The icon that will be shown in the IconButton. |

Props from [IconButton](https://material-ui.com/api/icon-button) are also used.

### AppHeaderFillSpace

The `AppHeaderFillSpace` component is used to separate the left and right elements of the `AppHeader` children.
Basically everything after `AppHeaderFillSpace` is pushed to the right of the `AppHeader`.

#### Props

| Name              | Type          | Default value  | Description                                              |
|:----------------- |:------------- |:-------------- |:-------------------------------------------------------- |
| startIcon         | node          |                | Icon shown at the start of the action.                   |
| endIcon           | node          |                | Icon shown at the end of the action.                     |
| disableTypography | boolean       |                | Prevents children from being wrapped with `Typography`.  |

Props from [ButtonBase](https://material-ui.com/api/button-base) are also used.

### AppHeaderDropdown

`AppHeaderDropdown` can be used for dropdown menus or other dropdown content.

<Canvas>
    <Story id="stories-components-appheader-dropdown--appheader-dropdown" />
</Canvas>

#### Props

| Name           | Type                                                        | Default value | Description                                                                                                                                                                        |
|:-------------- |:----------------------------------------------------------- |:------------- |:---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| children       | node \| (closeDropdown: () => void) => node                 |               | Content or a function that returns the Content shown in `Popover`. The function should be used if the Dropdown should be closed e.g. when a button inside the Dropdown is clicked. |
| actionContent  | node                                                        |               | Content passed into the `AppHeaderAction`.                                                                                                                                         |
| dropdownArrow  | null \| (isOpen: boolean) => node                           |               | Prevents children from being wrapped with `Typography`.                                                                                                                            |
| popoverProps   | [PopoverProps](https://material-ui.com/api/popover)         |               | Props passed into `Popover`.                                                                                                                                                       |

Props from [AppHeaderAction](#appheaderaction) are also used.
